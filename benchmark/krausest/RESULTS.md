## Methodology

Standard js-framework-benchmark keyed operations. Each op: unmeasured prep,
then one measured `click()` — timed in-page from before the synchronous click
dispatch to a double `requestAnimationFrame` after it (script + style + layout +
frame commit). DOM state is asserted after every measured action.

## Caveats — read before quoting these numbers

- This is NOT the official js-framework-benchmark harness: no CPU throttling
  (official runs several ops at 4x-16x slowdown), in-page double-rAF timing
  instead of CDP timeline tracing (excludes raster/composite), and far fewer
  samples. Use it for relative ordering, not for cross-publication comparison.
- Headless Chromium; headed numbers differ slightly.
- `what` is built from the repo working tree (not the published npm package).
- The double-rAF wait imposes a frame-scheduling floor of roughly 8-10ms on
  every op (visible on select/swap/remove, whose script cost is <1ms). The
  floor is identical for all frameworks, but it compresses the vs-vanilla
  ratios on cheap ops toward 1 — treat differences under ~2ms as noise.
- Implementations follow each framework's official krausest entry idioms
  (React: memo + immutable updates; Solid/What: per-row label signals).
