# What Framework v0.11.0

React compat that actually runs React libraries, a fullstack scaffold that works out of the box, and a faster compiler. All 14 packages release together at `0.11.0`; the test suite grew from 1068 to 1300+ tests.

## Highlights

### React compat, reworked

The compat layer now implements **real React hook semantics**: `useState`, `useReducer`, `useMemo`, and friends return plain values, and a compat re-render runtime re-executes compat components on state change — the contract React libraries actually depend on. CJS interop was fixed so CommonJS-built libraries (react-select, @emotion/*) load too.

Six real libraries are verified in CI on every push — failing, not skipping, if the fixture is missing:

- zustand
- @tanstack/react-query
- react-hook-form
- react-hot-toast
- @headlessui/react
- framer-motion (browser-verified)

### Fullstack scaffold works

`npm create what@latest my-app -- --fullstack` now produces an app that installs, builds, serves, and hydrates: file-routed SSR pages, a server action, and origin-first ISR via `server.js`. A scaffold smoke gate in CI builds both templates from local tarballs and asserts hydration in a real Chromium on every push.

### Compiler & runtime performance

- Branch memoization for conditional JSX — ternary/`&&` arms compile to cached templates.
- Specialized per-attribute setters replace the megamorphic `setProp` hot path.
- Single-evaluation mount path, zero-dependency effect release pool.
- Tree-shakeable compiled output.

Result: ~1.06x vanilla JS on krausest-style keyed list benchmarks.

### Deploy adapters verified

The node / static / vercel / cloudflare adapters from 0.10 are now covered end-to-end by tests.

## Security

- Blocked backslash open-redirect variants (`/\evil.com`) in server redirects.
- Action request bodies size-capped on the fetch-handler path.
- CSRF cookie issued with the `Secure` attribute.

## Fixes

- Reactive fragment expression children update correctly.
- `setStyle` clears stale keys when a style object loses properties.
- `what-devtools-mcp`: graceful `EADDRINUSE` handling, quieter MCP client, devtools test-runner hang fixed.
- ESLint presets (`plugin:what/recommended`) resolve correctly; guardrails wired; `what_eval` denylist hardened.

## Docs & honesty pass

- Fabricated benchmark numbers removed; bundle-size and compat claims standardized to measured numbers.
- whatfw.com is now itself built with What (SSG, 40 pages).
- `llms.txt` / `llms-full.txt` shipped.

## Upgrading

All `what-*` packages, `create-what`, and `eslint-plugin-what` move to `0.11.0` together — bump them as a set:

```bash
npm install what-framework@^0.11.0 what-compiler@^0.11.0
```

The react-compat hook-semantics change is breaking **within the compat layer only** (`what-react`): hooks now return values instead of signal accessors. Idiomatic What code (`signal()`, `computed()`, `effect()`) is unaffected.
