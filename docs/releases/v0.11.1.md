# What Framework v0.11.1

A focused follow-up to v0.11.0 from a full public-surface + engineering audit. All 14 packages release together at 0.11.1 (fixed-group). **No API changes — this is a safe upgrade from 0.11.0.**

## Fixed
- **`what-framework-cli init` now scaffolds a working app.** It previously generated only a `package.json` + config whose scripts pointed at a `what` bin no dependency provided. It now delegates to the create-what scaffolder; the printed next steps work end to end.
- **The fullstack template no longer serves server-only source over HTTP.** `src/db.js` and `src/actions/**` were fetchable in dev *and* prod. Static serving is now deny-by-default — only client assets resolve, server-only modules 404 (verified against path-traversal/encoding attacks).
- **Production bundles no longer run in dev mode in the browser.** `__DEV__` defaulted to `true` whenever there was no `process` global (every browser), shipping dev warnings and an internal `template()` XSS warning to production sites. It now resolves production-safe and prod builds strip every dev branch.
- **No-JS form submission works.** SSR forms now carry the `_action`/CSRF inputs the server needs, and the server accepts the CSRF token from the form body.
- **Unknown routes return real 404s** with `no-store` instead of cacheable soft-404s.

## Changed
- **Playground teaches JSX** (compiled in-browser by the real What compiler) instead of hand-written `h()` calls, with a "view compiled output" toggle.
- **Docs & marketing sites**: working docs search (`/` + Cmd-K), honest react-compat hero, build-time version badges, favicons, and one consistent version across every surface.
- **Release hygiene is mechanical** — version bumps now sweep all version strings and the CHANGELOG/SECURITY window so versions can't drift.

## Build & publish
- **`what-core` npm payload shrank 4.2 MB → 423 KB** (clean dist, ship only minified bundles, no sourcemaps in the tarball); added a size-budget CI gate.
- Compiler Vite 8 (`oxc`) support, `--help` on both MCP bins, and a devtools-mcp console-noise fix.
